Why I rebuilt my portfolio from scratch

Aug 02, 2026 · 4 min read

A short note on why I tossed the React template I started with and hand-wrote everything in HTML and CSS instead.

I started this site three times. The first attempt was Next.js with Tailwind — modern, fast to set up, and within a week I'd lost interest in writing anything on it. The second was a Notion-style template I bought. It looked fine. It also looked like everyone else's site.

The third attempt was this one. Plain HTML, plain CSS, no framework, no build step. I can edit any page in any text editor and push. The whole site weighs less than one Tailwind icon library.

What changed

The thing that flipped for me was reading old personal sites on the web — sites that have been around since 2005, still readable, still themselves. The common thread wasn't aesthetic. It was that the people who maintain those sites can actually maintain them. No `npm install`, no broken dependencies, no migration from one framework to the next every two years.

I wanted something that would still feel right when I came back to it in a year. Something I'd want to add to, not migrate.

The medium is the message, and the medium of a personal site is the act of maintaining it.

The constraints

I gave myself a few rules going in. Some of them I broke. Most of them held.

  • System fonts only (Georgia and Courier New). No web font loads.
  • No JavaScript on the homepage. The site works fully without it.
  • Every page printable. @media print styles from day one.
  • One stylesheet. No CSS-in-JS, no preprocessors, no Tailwind.

What I'd do differently

The Mac OS 8 scrollbar took three evenings I didn't budget for. Worth it — but I should have done it last, not somewhere in the middle. Lesson noted.

If you're thinking about doing the same thing: don't overthink the first version. Mine looked terrible for the first week. The point isn't to ship a beautiful site. It's to ship a site you can keep.
